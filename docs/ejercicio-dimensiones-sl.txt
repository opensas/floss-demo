¿Cómo las tecnologías y prácticas que vimos en la demostración nos ayudan a sacar provecho de las diferentes dimensiones del software libre?

A continuación tiene una descripción de ciertas características de las tecnologías y metodologías de trabajo utilizadas en la demostración del desarrollo de la solución libre.

(en este ejercicio tomamos github y git tan sólo por tratarse de las soluciones más utilizadas, existen otras como SourceForge, BitBucket, Launchpad, Google Code Hosting o Mercurial Bazaar, etc.)

Relacione cada uno de estos puntos con una o más de las siguientes dimensiones del softwate libre:

estudiar como funciona - libertad 1
distribuir el programa - libertad 2
mejorar el programa - libertad 3

git es un sistema de control de código fuente distribuido. Cada repositorio git es un repositorio completo y autónomo, con el historial completo de cambios, que puede ser usado con independencia respecto a cualquier otro repositorio central o incluso sin acceso a la red.

github es un servicio de hosting para desarrollo de proyectos de software. permite acceder a los proyectos libres, modificarlos y aportar las mejoras. Algunos lo llaman el Facebook de los desarrolladores.

github contiene actualmente casi 4 millones de repositorios, muchos de los cuales pueden ser consultados y utilizados para proyectos libres.

el dueño del repositorio de github, agrega colaboradores a los cuales les da permiso de escritura

el desarrollador de software libre busca en github si hay algún proyecto que le sea de utilidad

los cambios que realiza el desarrollador de software libre de nuestro ejemplo, son visibles para quien lo contata, y para todo el público, en cuanto los sube al repositorio de github

cada cambio que es subido al repositorio de github, queda relacionado con la persona que lo realizó, cada línea de código que se modificó y el ticket relacionado

al incluir hacer uso de un proyecto libre, hay que incluir la información de copyright y la licencia utilizada por el proyecto

cualquier usuario de github puede hacerse su propia copia de un repositorio (forkear) y realizar modificacion, y devolver los cambios al dueño del repositorio para que considere aceptarlos.

al ver el código fuente pudimos ver cómo estaba hecho el programa, qué hacía y qué librerías utilizaba.

al ver el código, un programador pudo rápidamente darse cuenta cómo agregar soporte para otro idioma

el acceso al código fuente, y la manera abierta y colaborativa en que fue desarrollado, hacen casi imposible la existencia de "prestaciones" ocultas (backdoors, spyware, etc.)

en el repositorio git y en github no sólo queda el código fuente, también tenemos el historial de cambios, los tickets, las discusiones y la documentación y demás artefactos generados durante el proceso de desarrollo.

al tratarse de un proyecto libre, cualquier persona puede simplemente descargarse el proyecto y comenzar a utilizarlo sin ningún otro trámite, tal como hizo nuestro desarrollador con la librería traductor_lib.

cualquier desarrollador con el conocimiento necesario, podría dar soporte y desarrollar nueva funcionalidad para el sistema "traductor".
